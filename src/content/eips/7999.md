EIP-7999 is the **more ambitious** alternative to the minimal set. It is more complex —
it touches the fee market itself rather than just a few gas constants — but it buys a much
larger set of benefits. (This page tracks the latest revision in
[ethereum/EIPs#11835](https://github.com/ethereum/EIPs/pull/11835).)

## The idea

Today a user juggles separate fee markets: EIP-1559 for execution, EIP-4844 for blobs,
floor rules for calldata. EIP-7999 replaces that with a **single multidimensional fee
market**. Each transaction specifies one `max_fee` in ETH — the maximum it will pay for
inclusion — and the protocol treats that ETH as fungible across every resource it consumes,
guaranteeing it can pay for all of them.

## Four metered resources

Gas becomes a 4-element vector `[execution, blob, data, state]`:

| Resource | What it covers | Notes |
| --- | --- | --- |
| **Execution** | Traditional EVM compute | Target : limit ratio of 2:1 |
| **Blob** | EIP-4844 blobs | Block blob count; unconditional under EIP-7805 |
| **Data** *(new)* | Tx content bytes (EIP-8131) + BAL bytes (EIP-8279) | `16` gas/content byte; its own base fee |
| **State** *(new)* | State growth (EIP-8037 metering) | **Target only, no hard cap**; own base fee |

Block headers carry `gas_limits` for execution/blob/data and `gas_used` / `excess_gas`
vectors that also include state. A single `BASE_FEE_UPDATE_FRACTION` drives every
resource, with excess normalized by each resource's limit (or its *target*, for state,
which has no limit).

## New opcodes

- **`DATABASEFEE` (0x4b)** — current block's data base fee
- **`STATEBASEFEE` (0x4c)** — current block's state base fee

Both follow the EIP-7516 pattern established for the blob base fee.

## Why it's worth the extra complexity

1. **One fee for users.** A single `max_fee` mirrors how people actually think about paying
   for Ethereum, instead of three coupled fee markets.
2. **Decoupled throughput.** Splitting data into its own resource raises data throughput
   *and* lowers its cost while leaving EVM gas free for everything else — congestion in one
   dimension no longer taxes the others.
3. **Exact long-run state control.** The state dimension equilibrates around a target
   instead of a hard cap, giving precise control over state growth while still allowing
   temporary spikes many times today's gas limit.
4. **Smoother fees.** Normalizing excess by limit/target makes the base fee respond
   smoothly whenever any limit is adjusted.
5. **Censorship resistance preserved.** Data and state are *unconditional* under EIP-7805
   (FOCIL), so inclusion-list transactions can't be censored by resource stuffing.
6. **No rebate secondary markets.** Block-level pricing avoids the secondary markets that
   transaction-level floor pricing (EIP-7623) creates — which EIP-7999 deprecates.

## Minimal set vs. EIP-7999

The minimal set (8131 + 8279 + 8311) is the smallest safe path to 450M. EIP-7999 reaches
the same scaling goal **and** restructures the fee market so future scaling is cheaper and
state growth is precisely controllable — at the cost of a deeper, riskier protocol change.
It subsumes the data-pricing work of 8131/8279 as its "data resource."
